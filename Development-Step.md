### Database Schema to Create in Supabase:

#### Table: `app_settings`
```sql
CREATE TABLE app_settings (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  user_id TEXT UNIQUE NOT NULL,  -- stores auth.uid()::text
  current_theme TEXT NOT NULL DEFAULT 'light',
  theme_colors JSONB NOT NULL DEFAULT '{...default colors...}',
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
```

#### Table: `notes`
```sql
CREATE TABLE notes (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id TEXT NOT NULL,
  title TEXT NOT NULL,
  content TEXT NOT NULL,
  created_at TIMESTAMP DEFAULT now(),
  updated_at TIMESTAMP DEFAULT now(),
  deleted BOOLEAN DEFAULT false
);
```

#### Table: `folders`
```sql
CREATE TABLE folders (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id TEXT NOT NULL,
  name TEXT NOT NULL,
  parent_id UUID REFERENCES folders(id),
  created_at TIMESTAMP DEFAULT now(),
  updated_at TIMESTAMP DEFAULT now(),
  deleted BOOLEAN DEFAULT false
);
```

#### Table: `files`
```sql
CREATE TABLE files (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id TEXT NOT NULL,
  folder_id UUID REFERENCES folders(id),
  telegram_file_id TEXT NOT NULL,
  file_name TEXT NOT NULL,
  file_type TEXT NOT NULL,
  file_size INTEGER NOT NULL,
  uploaded_at TIMESTAMP DEFAULT now(),
  deleted BOOLEAN DEFAULT false
);
```

#### Table: `theme_presets`
```sql
CREATE TABLE theme_presets (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id TEXT NOT NULL,
  name TEXT NOT NULL,
  preset_key TEXT NOT NULL,
  colors JSONB NOT NULL,
  created_at TIMESTAMP DEFAULT now(),
  is_default BOOLEAN DEFAULT false
);
```

#### Table: `message_metadata`
```sql
CREATE TABLE message_metadata (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id TEXT NOT NULL,
  telegram_message_id TEXT NOT NULL,
  content TEXT NOT NULL,
  is_file BOOLEAN DEFAULT false,
  created_at TIMESTAMP DEFAULT now()
```

```javascript

// Routes:
//   /login -> Login page

//   /notes -> Notes page (protected)
//   /explorer -> File Explorer (protected)
//   /settings -> Settings page (protected)
//   / -> Redirect based on auth status
```

### Key Functions to Implement:
1. **Login**: `supabase.auth.signInWithPassword({ email: VITE_EMAIL, password })`
2. **Logout**: `supabase.auth.signOut()`
3. **Check Session**: `supabase.auth.getSession()` (runs once on mount)
4. **Auth State Listener**: `supabase.auth.onAuthStateChange()` (auto-syncs SIGNED_IN/SIGNED_OUT/TOKEN_REFRESHED)
5. **Change Password**: Re-authenticate + `supabase.auth.updateUser({ password })`
6. **Get User ID**: `supabase.auth.getUser()` returns user.id

### Supabase RLS Policies:
```sql
-- All tables use this pattern:
CREATE POLICY "Users can view own data" ON table_name
    FOR SELECT USING (user_id = auth.uid()::text);
CREATE POLICY "Users can insert own data" ON table_name
    FOR INSERT WITH CHECK (user_id = auth.uid()::text);
CREATE POLICY "Users can update own data" ON table_name
    FOR UPDATE USING (user_id = auth.uid()::text);
CREATE POLICY "Users can delete own data" ON table_name
    FOR DELETE USING (user_id = auth.uid()::text);
```

### Testing Points:
- Login with correct password
- Login with wrong password (shows error)
- Logout and login again
- Session persists on page reload
- Theme preferences persist across sessions
- No infinite re-renders (Context prevents duplicate state)
